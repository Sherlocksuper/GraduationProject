## 第 3 张：系统方案与关键机制

**页面标题**  
多智能体协同方案与关键机制

**页面正文（逐条放入 PPT）**  
1. 采用 ReAct 思路：推理与行动交替执行，先收集证据，再组织结论。  
2. 角色分工：Coordinator 负责编排；Planner 负责规划；Researcher 负责检索；Fetcher 负责抓取；Reader 负责证据提炼；Writer 负责写作；Critic 负责质量检查。  
3. 任务状态机：created → planning → collecting → reading → writing → reviewing → done/failed。  
4. 关键产物：`plan.json`、`sources.json`、`notes.json`、`report.md`、`critic_report.json`。  
5. 关键机制：异常重试、失败降级、质量审查三者组合，保障任务完成率与结果可解释性。

**图片占位符**  
[placeholder: trace2.png]

**讲稿（90 秒）**  
这页重点讲系统“怎么做”。我没有采用单模型一次性生成，而是采用多角色分工流水线。这样做的好处是每一步职责明确，出了问题能快速定位。比如检索阶段失败，不会影响已完成的规划；抓取阶段失败可降级为 snippet，不会导致全任务中断。最终输出不仅有报告，还有 plan、sources、notes 和 critic 报告，便于老师检查过程与结果的一致性。

## 第 4 张：中期实现效果（证据链 + 报告结果）

**页面标题**  
中期实现效果：从证据到报告

**页面正文（逐条放入 PPT）**  
1. 检索阶段使用 Tavily 返回结构化结果，trace 中可见 query、provider、URL。  
2. 抓取阶段正文优先，若遇反爬/超时自动降级 snippet，保证流程不断链。  
3. 阅读与写作阶段按“claim/support/url”组织证据并分节写作，每节附来源。  
4. Critic v0 对来源数量、域名多样性、硬断言风险进行最小检查。  
5. 当前系统可输出“可阅读报告 + 可回溯证据 + 可复盘过程”三类结果。
