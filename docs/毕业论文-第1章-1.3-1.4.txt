1.3 相关技术

本节对课题涉及的关键技术作简要说明，为后文需求分析、系统设计与实现奠定概念基础。整体技术路线围绕“对话式人机交互 + 服务端编排 + 多智能体协作机制”展开，并辅以检索增强与工具调用等常用工程手段。

1.3.1 大语言模型与对话应用

大语言模型指在海量文本上预训练得到的通用语言模型，经指令微调与人类反馈对齐等后训练技术后，往往具备较强的指令遵循与多轮对话能力。在智能聊天系统中，模型通常承担意图理解、策略选择、内容生成与格式控制等职责，其输入一般由系统提示、会话历史与当前用户消息共同组成。

对话应用强调上下文连续性与交互成本：一方面需要控制上下文长度以兼顾成本与时延；另一方面需要在多轮交互中保持主题一致、避免跑题或自相矛盾。因此，工程上常结合截断、摘要、窗口管理与外部记忆等策略，以提升长会话稳定性。对本课题而言，大语言模型是各智能体“语言能力”的共同底座，而聊天界面则是用户感知系统能力的主要入口。

1.3.2 ReAct 范式与工具调用（Tool Calling）

ReAct 范式将“推理”和“行动”交替组织：模型在思考后选择是否调用外部工具，系统执行工具并返回观察结果，模型再基于观察继续推理或结束并给出答复。与仅输出一次性结论相比，该方式把部分关键信息交给可验证的外部过程，有利于提升复杂任务的可控性与可解释性。

工具调用（Tool Calling）是工程落地的重要接口形态：通过预定义工具名称、参数模式与返回结构，服务端可对模型输出进行解析、校验与执行，并将结果以统一格式回填到对话上下文中。对本课题而言，工具调用既是实现计算器、时间查询等确定性能力的手段，也可作为多智能体协作中“执行型角色”与外部世界交互的桥梁。

1.3.3 检索增强生成（RAG）与向量检索

检索增强生成（RAG）指在生成前先从知识库或文档集合中检索与问题相关的文本片段，再将其作为证据注入提示词，引导模型基于材料作答。其动机在于缓解纯参数记忆带来的事实偏差，并支持引入用户私有资料或领域文档，使回答更贴近特定场景。

向量检索是 RAG 的常见实现路径：将文本分块后通过嵌入模型得到向量表示，查询时把用户问题同样嵌入为向量，通过相似度度量（如余弦相似度）检索 Top-K 片段。工程上还需关注分块粒度、去重、阈值过滤与多库合并等问题，以在召回率与噪声控制之间取得平衡。对本课题而言，RAG 可作为聊天系统中“资料型智能体”或资料准备环节的支撑技术，用于增强回答依据。

1.3.4 多智能体与分阶段任务编排思想

多智能体系统强调多个具备不同职责或不同提示策略的智能体协同完成目标。与单智能体“端到端一次生成”相比，多智能体结构更利于把复杂任务拆解为规划、检索、执行、写作、检查等子过程，并由协调逻辑决定调用顺序、合并策略与失败降级方式。

分阶段任务编排指将任务明确划分为若干阶段，每阶段产出结构化中间结果，供下一阶段消费。该思想与多智能体协作天然契合：不同阶段可由不同智能体负责，也可由同一底座模型在不同系统提示下扮演不同角色。对本课题而言，“智能聊天”是外在形态，“多智能体协作与编排”是内在机制，两者结合有助于在统一会话体验下提升复杂任务完成度，并形成可记录的过程信息。

1.3.5 Node.js 与 Express 服务端技术

Node.js 是基于 Chrome V8 引擎的 JavaScript 运行时，适合实现 I/O 密集型的 Web 服务与异步并发处理。Express 是常用的轻量级 Web 框架，提供路由、中间件与请求响应抽象，便于快速搭建 REST 风格接口并集成鉴权、日志与错误处理等横切能力。

对本课题而言，采用 Node.js 与 Express 有利于前后端语言栈统一，降低开发与部署成本，并便于将大模型调用、工具执行、检索与任务编排等模块以服务端组件形式组织，从而把智能体协作逻辑与前端展示解耦。

1.3.6 React 与 Vite 前端技术

React 是组件化用户界面库，通过状态驱动视图更新，适合构建聊天窗口、会话列表、表单与复杂交互页面。组件化有助于将消息渲染、输入区、加载态与错误提示等拆分为可维护模块。

Vite 是现代化前端构建工具，开发阶段借助原生 ES 模块与快速热更新提升调试效率，生产构建则输出优化后的静态资源。对本课题而言，React 与 Vite 的组合能够支撑响应式聊天界面与后续功能扩展，并与服务端接口通过 HTTP 交互形成清晰的前后端边界。

1.4 本文的主要研究内容

本文围绕“基于多智能体的智能聊天系统”开展研究与实现，主要工作包括以下方面。

第一，分析智能聊天系统在多轮对话、复杂任务与可解释执行等方面的需求，明确多智能体协作在系统中的定位与边界，给出功能性与非功能性需求描述。

第二，给出系统总体设计方案，采用前后端分离架构组织模块，设计会话与消息等核心数据模型，并规划多智能体协作与编排流程在服务端中的落点，使聊天交互与后台执行结构一致。

第三，实现以对话为主入口的智能聊天系统：支持用户认证与会话管理，支持多轮对话；实现 ReAct 式工具循环，使模型在需要时发起工具请求并由服务端执行后回填观察结果；实现知识库检索增强对话能力，将检索片段作为上下文证据参与回答生成；实现分阶段的研究型任务编排作为复杂场景下的增强能力，并记录关键过程信息以便分析与展示。

第四，对主要功能进行测试与验证，从正确性、稳定性与交互体验等角度给出结果说明，并总结不足与改进方向。

以上内容构成了本文的研究范围与章节展开线索，后文将依次展开需求分析、系统设计、系统实现与测试总结。

1.5 章节安排

本章在前文中阐述了课题背景与研究目的、意义，综述了国内外在多智能体对话及相关技术方向的研究与应用现状，介绍了大语言模型对话、ReAct 与工具调用、检索增强生成、多智能体与分阶段编排，以及 Node.js、Express、React、Vite 等实现技术，并概括了本文的主要研究内容。各章安排如下。

第2章进行系统需求分析。首先说明分析范围与原则，对系统目标用户与使用场景进行描述；结合用户侧使用流程给出业务流程说明，并从用例角度对注册登录、多会话聊天、知识库管理、检索增强对话、深度研究任务及过程查看等功能需求进行分析；最后从安全性、可靠性、性能、易用性、可解释性与异常处理等方面归纳非功能性需求。

第3章进行系统设计。在总体架构上采用前后端分离与 B/S 模式，明确服务端与外部大模型及检索能力的关系；在功能上划分认证与会话、智能对话与工具编排、知识库与 RAG、深度研究管线及过程追踪等模块；完成核心实体与数据结构设计，并对各模块的关键流程与接口交互进行说明；同时补充错误处理、并发控制与界面交互等非功能性设计要点。

第4章介绍系统实现。围绕用户侧主要功能展开，分别说明认证与会话持久化、对话服务与 ReAct 工具循环、知识库分块与向量检索及对话注入、深度研究各阶段逻辑与 Trace 与产物组织等实现要点，并结合前端界面说明聊天、知识库管理与研究进度展示等交互实现，辅以关键界面截图帮助读者建立直观认识。

第5章进行系统测试。说明测试目的、方法与范围，按模块给出主要功能测试用例与结果，并对安全性、性能或稳定性等非功能性内容进行验证与小结。

第6章对全文工作进行总结，归纳已取得的成果与不足，并对后续在多智能体策略优化、评测体系与安全合规等方面的改进进行展望。
