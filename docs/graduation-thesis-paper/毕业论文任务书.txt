一、立题的目的和意义
--------------------------------------------------------------------------------

随着大语言模型与生成式人工智能的普及，智能对话已成为学习、办公与信息获取的重要形态。单一模型一次性生成答案的方式，在复杂问题、私域知识接入与过程可解释性方面存在不足。多智能体协作与分阶段任务编排，可将规划、检索、写作与检查等职责拆分，在统一聊天入口下提升任务完成度与可追溯性；结合检索增强生成（RAG）与工具调用范式，有利于在工程上平衡效果、成本与可控性。

本课题旨在设计并实现一套基于多智能体协作思想的智能聊天系统，集成用户认证、多会话对话、知识库管理与检索增强、深度研究型长流程任务及过程展示等功能，形成可运行、可演示、可扩展的原型。通过本课题，可综合运用 Web 全栈开发、数据库与外部模型服务对接等能力，为毕业后从事智能应用开发积累实践经验；论文工作则训练需求分析、系统设计、实现与测试的完整文档化能力。

--------------------------------------------------------------------------------
二、技术指标与主要内容
--------------------------------------------------------------------------------

（一）技术指标（可按答辩前实际达标情况勾选或量化）

1. 功能完整性：实现用户注册登录（含邮箱验证、验证码登录等能力中的主要路径）、多会话聊天、会话与消息持久化；支持知识库集合与分块管理、会话绑定知识库及检索增强对话；支持深度研究任务创建、状态查询、报告与过程信息展示。
2. 运行环境：服务端 Node.js 18+；前端现代浏览器可访问；数据持久化采用 SQLite 及必要的本地文件存储；可配置接入大语言模型与嵌入、联网检索等外部服务。
3. 可靠性：主要接口具备参数校验与鉴权；对模型调用失败、研究任务异常等具备错误提示或降级策略；同一会话深度研究防并发等控制逻辑可用。
4. 文档与可维护性：提供与实现一致的需求、设计、实现与测试说明（论文正文）；代码结构模块化，便于答辩演示与后续迭代。

（二）主要内容

1. 查阅国内外与多智能体对话、ReAct、RAG、深度研究原型相关的资料，完成开题与需求分析。
2. 完成系统总体设计与数据库/存储设计，划分认证与会话、对话与编排、知识库与 RAG、深度研究、前端展示等功能模块。
3. 实现前后端分离系统：Express 后端 API、React 前端界面、SQLite 业务数据存储及用户数据持久化方案。
4. 完成与大语言模型及可选嵌入、检索服务的对接；实现简单对话、检索增强对话及深度研究管线的核心路径。
5. 开展系统测试并撰写毕业论文，准备答辩材料（演示环境、截图与核心用例说明）。
