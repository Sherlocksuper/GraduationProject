第2章  系统需求分析

2.1  引言

本章从「谁能用、做什么、做到什么程度」出发，对基于多智能体的智能聊天系统做需求规约：侧重聊天主场景下的认证与会话、可选的知识库增强、以及长流程深度研究三类能力，并补充必要的非功能约束。表述上配合用例图与用例表，为第3章设计与第4章实现划定边界。

2.2  系统需求描述

系统面向已注册且完成必要核验的登录用户，B/S 架构，浏览器访问前端，服务端负责鉴权、会话与消息持久化、对接大模型与可选嵌入/检索服务，并在深度研究场景下执行分阶段编排与过程记录。

业务流程主线：登录，选择或新建会话，发送消息。在此基础上，若会话绑定了知识库，则回答前自动检索片段注入上下文；若本轮走深度研究，则后台执行规划—资料—写作—检查等阶段，用户可查看状态、产物与 trace。如图2-1。

如图2-2，注册用户侧功能可归纳为五个部分：

认证与账号：提交注册信息、邮箱验证、密码登录与验证码登录、登出、忘记密码与令牌重置、维护个人资料与修改密码。

会话管理：创建新会话、在多个会话间切换、删除会话、查看会话列表与会话内历史消息。

会话内智能对话：在选定会话中输入问题并发送、查看助手回复与多轮上下文；在需要时扩展为 ReAct 工具循环（由模型发起工具请求、服务端执行并回填观察直至给出答复）。

知识库与检索增强：新建与删除知识库集合、维护分块与入库、将会话绑定或解绑一个或多个知识库；在对话中按问题做向量检索、合并多库命中、将片段注入上下文并支持相似度门槛控制。

深度研究与过程查看：在对话中发起深度研究任务、获取任务标识并轮询状态；查看规划与中间产物、阅读最终报告；查看执行轨迹 trace 以理解分阶段编排过程。

以下按功能域给出用例表，与图2-2对应。

覆盖注册与邮箱验证、密码/验证码登录、登出、忘记密码、资料与密码维护；安全约束见 2.4。

多会话的创建、切换、历史查看与单轮/多轮问答；可按需走 ReAct 式工具循环（模型请求工具—服务端执行—回填观察直至答复）。

表2-3  会话内智能对话用例描述
用例内容	说明
用例名称	会话内发送消息并获取回复
角色	注册用户
简要说明	用户在指定会话中提问并获得系统生成的助手回复
前置条件	用户已登录；存在可用会话或用户新建会话成功

4.2. 工具执行失败，系统记录原因并尽量给出可理解提示

2.4  非功能性需求

运行与交付：Node.js 服务端 + 现代浏览器前端；具体机器与 IDE 定稿时补一行即可。

性能：普通对话在常用网络下可接受时延内返回；检索与研究允许更久，但须有加载/进度反馈。

可靠：模型与外部接口失败时有重试、降级或明确报错；会话与任务数据持久化，避免一次失败丢全量上下文。

易用：聊天、知识库、研究三类入口可区分；长文与 Markdown 可读；研究过程可感知。

安全：登录后访问；接口按用户隔离资源；令牌与口令不当面长期明文；验证码类接口限频。

可追溯：深度研究保留阶段级 trace，便于复核与论文分析（与第3章设计衔接）。

2.5  本章小结

本章首先依据注册用户的使用流程梳理了系统业务主线，并给出业务流程图（图2-1）；再从用户用例图（图2-2）及用例表（表2-1～表2-5）出发，归纳了认证与会话、智能对话与工具增强、知识库与检索增强对话、深度研究任务与过程查看等功能需求；最后从运行环境、性能、可靠性、易用性、安全与可追溯等方面分析了非功能性需求，为后续的系统设计做准备。
